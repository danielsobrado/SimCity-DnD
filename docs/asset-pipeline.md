# Static GLB asset pipeline

The settlement editor loads production object visuals from GLB files while retaining procedural models as a fail-safe.

## Authoring contract

- Format: glTF 2.0 binary (`.glb`).
- Coordinates: Y-up.
- Scale: one authored unit equals one logical map cell before the runtime tile-size scale is applied.
- Pivot: centered on the footprint in X/Z, with the lowest visible geometry at Y = 0.
- Materials: embedded PBR materials. External texture files are not accepted by the initial pipeline.
- Geometry: static meshes only. Skinned meshes and animation are intentionally rejected for settlement props.
- Bounds: X/Z geometry must remain within the configured footprint with a small visual-overhang allowance.
- Paths: assets live below `public/assets/models` and are referenced from `config/objects.yaml`.

## Object definition

```yaml
- key: cottage
  model: cottage
  asset:
    path: "assets/models/settlement-core.glb"
    node: "cottage"
    scale: 1
    rotationY: 0
    offset: [0, 0, 0]
  footprint: { width: 2, depth: 2 }
```

`model` selects the procedural fallback. `asset` selects the production GLB. Placement rules and saves use the object key and footprint, never the rendered model.

## Runtime behavior

1. The editor starts immediately with procedural fallback geometry.
2. `ObjectAssetRepository` loads each GLB pack once through `GLTFLoader`.
3. The configured top-level node is selected from the shared pack.
4. Visible static meshes are converted into instanced render parts.
5. Geometry is validated against the configured footprint and ground pivot.
6. Successful assets replace their fallback renderer without changing placed-object IDs.
7. Missing or invalid assets leave the procedural fallback active and report the failure.

## Generation

The placeholder pack is generated by `scripts/generate-placeholder-models.py` using only Python's standard library. The `Build GLB Assets` workflow regenerates and commits the pack whenever its source or object configuration changes.

Manual regeneration:

```bash
npm run generate:assets
```

## Validation

Run:

```bash
npm run validate:assets
```

The validator checks:

- Every object has a GLB pack, node selector, and procedural fallback.
- Asset paths stay inside `public`.
- GLB magic, declared length, version, and JSON chunk.
- glTF 2.x metadata, scenes, nodes, and meshes.
- Every committed GLB is referenced by the object catalog.

The full gate is:

```bash
npm run verify
```
